# Финальный проект модуля Python по программе Аналитик данных Simultative

## Структура Python-скрипта
1. Класс `PGConnection` для подключения к базе данных PostgreSQL. Экземпляр класса создавать не нужно. Класс содержит статический метод `get_pg_connection`.
2. Класс `ApiResponseReader` для получения данных по API. Нужно создать экземпляр класса с атрибутами `url` и `params` для доступа к API. Метод `get_response` получает ссылку на экземпляр класса и возвращает массив "сырых" данных.
3. Класс `ApiResponseProcessing` для обработки "сырых" данных. Нужно создать экземпляр класса с атрибутами `data` и `tasks_stat` для доступа к API. Метод `pars_response` получает ссылку на экземпляр класса и массив необработанных данных, и возвращает массив данных в нужном формате.
4. Класс `ImportData` для загрузки данных в базу данных PostgreSQL. Экземпляр класса создавать не нужно. Класс содержит только один статический метод `import_data`.
5. Класс `LogFilesCleaner` для удаления старых файлов логов.  Экземпляр класса создавать не нужно. Класс содержит статический метод `remove_old_log_files`, который принимает количество дней за которые нужно сохранить файлы логов (по умолчанию 3) и очищает файлы из директории logs.
6. Класс `GoogleSheetImportReport` для загрузки данных в Google-таблицу. Экземпляр класса создавать не нужно. Класс содержит статический метод `import_data_google_sheet`, который принимает данные в виде списка списков со статистикой попыток решения задач и загружает данные в Google таблицу. [Ссылка на таблицу с отчетом](https://docs.google.com/spreadsheets/d/1UAXxfNoAaBmKzo2GghuZOKGI9DQYKzSx7zp_wyP09Gc/edit?gid=0#gid=0)
7. Класс `SendEmailReport` для отправки отчета на Email. Экземпляр класса создавать не нужно. Класс содержит статический метод `send_email`, который принимает данные в виде списка списков со статистикой попыток решения задач, формирует email и отправляет его адресату.

## Запуск проекта
1. Сделать клон проекта.
2. Создать локальную базу данных в СУБД PostgreSQL.
3. Создать файл `credentials.py` в корне проекта и прописать значения параметров:
    - `client` = "клиент для доступа к API". Берем из текста задания.
    - `client_key` = "Ключ для доступа к API". Берем из текста задания.
    - `user_db` = "Имя пользователя БД". 
    - `password_db` = "Пароль доступа к БД". 
    - `host_db` = "localhost". Если работаете с удаленной базой, то укажите адрес сервера БД.
    - `port_db` = 5432
    - `sender_gmail` = "Email адресата"
    - `smtp_server_gmail` = "smtp.gmail.com". Для другого почтового сервиса нужно указать другой адрес сервера.
    - `smtp_port_gmail` = 465. Для другого почтового сервиса нужно указать другой порт.
    - `password_gmail` = "Ваш пароль приложения для авторизации в Google". Как получить смотрите п.6
4. [Создать сервисный аккаунт в Google для работы с Google Sheets API](https://support.google.com/workspacemigrate/answer/10839762). Обязательно включите Google Sheets API.
5. [Сгенерировать файл с ключами от учетной записи сервисного аккаунта.](https://developers.google.com/workspace/guides/create-credentials#create_credentials_for_a_service_account) Назвать его `creds.json` и поместить в корень проекта.
6. [Сгенерировать пароль приложения Google для отправки Email](https://support.google.com/accounts/answer/185833)
7. Запустить файл `etl_pipeline.py`
